# Client Job Quickstart

## Purpose

This is the fastest starting point for real small-to-medium client jobs.

Use this file before opening the full template system. It helps decide:

- what type of job this is
- which 2 to 3 templates to use
- what to ask in the first client call
- what privacy red flags to check
- what deliverable format to choose
- when client approval is needed
- whether the job can become a portfolio case

This checklist is for AI PM, IP building, digital marketing, landing page / prototype, AI workflow audit, and consulting work. It is not legal or compliance advice. Legal / compliance confirmation may be needed depending on the client, sector, data type, and jurisdiction.

# 2. First Client Call: 10 Questions

Ask these before quoting final scope or using AI tools.

```text
1. Goal
What do you want this job to achieve?

2. Audience
Who is the target audience or decision-maker?

3. Current materials
What materials already exist? Notes, posts, PDFs, screenshots, decks, transcripts, reports, website links?

4. Deadline
When do you need the first draft and final version?

5. Channel
Where will the output be used? IG, LinkedIn, website, proposal, internal report, landing page, email, presentation?

6. Approval owner
Who has final approval authority before delivery or publishing?

7. Forbidden topics / claims
Are there topics, claims, words, visuals, competitors, or promises that must be avoided?

8. Sensitive data involved
Does any material include personal data, student / youth / SEN / service user data, health, psychological, financial, identity, HR, legal, or private client information?

9. Expected output format
Do you expect Google Doc, Notion page, PDF, Canva deck, Google Sheet / Excel, Markdown, GitHub docs, or another format?

10. Budget / scope priority
If time or budget is limited, which deliverable is the highest priority?

11. Portfolio permission
After delivery, can this be used as a public case, anonymized case, internal-only case, or not used in portfolio at all?
```

Call output:

```text
Job type:
Selected templates:
Main deliverable:
Deadline:
Approval owner:
Privacy red flags:
Portfolio permission:
Next action:
```

# 3. Scope & Quote Guardrail

Confirm these boundaries before quoting or starting the work.

```text
Included deliverables:
- What exact outputs are included?
- How many posts, pages, slides, summaries, or versions?
- Is strategy included, or only content production?

Not included:
- Posting / publishing on client accounts unless explicitly agreed.
- Ad setup, community management, analytics reporting, or ongoing optimization unless explicitly agreed.
- Graphic design, video editing, landing page implementation, or account operation unless explicitly included.
- Performance guarantee, including guaranteed reach, sales, leads, conversion, or follower growth.

Revision rounds:
- How many revision rounds are included?
- What counts as minor wording changes?
- What counts as scope change?
- What happens after included revisions are used?

Client-provided materials and permissions:
- Client confirms they have the right to use provided text, images, videos, fonts, music, logos, screenshots, and brand assets.
- Canva / Freepik / stock image / music / font assets must stay within license scope.
- Competitor examples are for structure learning only, not direct copying.

Approval owner:
- Confirm who approves source material, draft wording, visual direction, claims, final delivery, and portfolio permission.
```

Quote rule:

```text
Do not quote only by task title. Quote by deliverables, revision rounds, approval owner, deadline, material risk, and what is not included.
```

# 4. Privacy Red Flags

Stop and check before using AI if any item below is present.

```text
[ ] Personal data: names, emails, phone numbers, addresses, IDs, photos, account IDs
[ ] Student / youth / SEN / service user data
[ ] Health / psychological / counselling / financial / identity / HR / legal data
[ ] Private client documents, contracts, strategy, pricing, internal reports
[ ] Screenshots with account names, private dashboards, analytics IDs, customer details
[ ] Copyrighted assets: paid course material, reports, images, music, fonts, logos, videos
[ ] Competitor content copying request
[ ] Platform login / account access request
[ ] Claims requiring legal / medical / financial / compliance confirmation
[ ] Client asks to upload raw files without anonymization or approval
[ ] Client wants guaranteed sales, conversion, reach, or performance result
```

Hard rule for NGO / education / youth / SEN / social service contexts:

```text
If the material involves minors, SEN, service users, counselling, psychological status, health status, case notes, or identifiable activity participants, do not upload original text, photos, recordings, transcripts, or identifiable case details to AI tools by default.

Use anonymized summaries or fake data first. Confirm organization policy, consent, and approval before using any real material.
```

If unsure, use safe fallback:

```text
Do not upload raw material.
Create a short anonymized summary.
Use fake or generic example data for AI drafting.
Apply the output manually to the real context after human review.
Ask for written approval before using sensitive or client-owned materials.
```

# 5. Small Job Shortcut

For small HK$1000–3000 jobs, do not open every template.

## Lightweight Path

```text
1. Use `docs/CLIENT_SAFE_AI_WORKFLOW.md`
   - Check data sensitivity.
   - Remove private details.
   - Confirm approval owner.

2. Use one production template:
   - Content job: `docs/CONTENT_REPURPOSING_TEMPLATE.md`
   - Landing page / prototype job: `docs/LANDING_PAGE_PROTOTYPE_BRIEF.md`
   - Audit job: `docs/CONSULTING_AUDIT_TEMPLATE.md`

3. Use `docs/PORTFOLIO_CASE_TEMPLATE.md` only after delivery if appropriate.
   - Do not prepare a public case unless permission is clear.
```

Small job minimum output:

```text
- short source summary
- privacy / risk note
- draft output
- approval checkpoint
- final deliverable
- optional portfolio note
```

Not every small job needs the full workflow. Use the smallest safe path that produces the required deliverable.

# 6. Deliverable Format Guide

| Output Type | Recommended Format | Notes |
|---|---|---|
| Short content pack | Google Doc / Notion page / Markdown | Best for captions, post ideas, hooks, CTA, carousel outline |
| Campaign proposal | Google Doc / Canva deck / PDF | Use deck if client needs presentation; use Doc if still in draft stage |
| Landing page brief | Google Doc / Notion page / Markdown / GitHub docs | Good for product thinking, page sections, copy direction, prototype scope |
| Workflow audit | Google Doc / Notion page / PDF | Include current workflow, pain points, AI opportunities, roadmap, risks |
| Portfolio case | Markdown / GitHub docs / Notion page | Use only fake, public, or approved anonymized material |
| Client meeting summary | Google Doc / Notion page / Google Sheet | Include action items, owners, deadlines, risks, next steps |
| Content calendar | Google Sheet / Excel / Notion table | Best for dates, channel, status, owner, CTA, approval status |
| Visual direction | Canva deck / Google Doc / PDF | Include moodboard notes, layout references, brand risks, asset needs |

Format rule:

```text
Draft stage: Google Doc / Notion / Markdown
Client presentation: Canva deck / PDF
Tracking work: Google Sheet / Excel / Notion table
Portfolio evidence: Markdown / GitHub docs / Notion page
```

# 7. Approval Checkpoints

Use these checkpoints before delivery or publishing.

```text
[ ] Source material approval
Client confirms the materials can be used for this job.

[ ] Draft wording approval
Client reviews wording, tone, CTA, and key message.

[ ] Visual / brand approval
Client approves visual direction, logos, images, layout references, and brand usage.

[ ] Claim / compliance approval
Client reviews claims that may involve legal, medical, financial, compliance, education, youth, SEN, or service-user context.

[ ] Final publish approval
Client confirms the final version can be published, delivered, or sent externally.

[ ] Portfolio permission approval
Client confirms whether the work can be used publicly, anonymously, internally, or not at all.
```

Approval record:

```text
Deliverable:
Version:
Approval owner:
Approval date:
Approved use: internal / client delivery / public publishing / portfolio
Restrictions:
Record location: email / message / signed approval / project document
```

Written or traceable approval is preferred for portfolio use, public publishing, account access, sensitive claims, and client-owned assets.

# 8. Stop / Escalate Conditions

Pause, reject, or ask for clarification if:

```text
[ ] Client asks for guaranteed performance results, such as guaranteed sales, leads, reach, or conversion.
[ ] Client asks to copy competitor content directly.
[ ] Client provides sensitive personal data without consent or clear approval.
[ ] Client asks to bypass platform rules, copyright rules, or account restrictions.
[ ] Client refuses human approval before publishing.
[ ] Scope is unclear but deadline is urgent.
[ ] Legal / medical / financial / compliance claims are required.
[ ] Account login or admin access is needed without written permission.
[ ] Client wants to upload raw confidential files to AI tools without review.
[ ] Client expects a prototype to behave like a production system without QA or maintenance scope.
```

Suggested response:

```text
This part needs clarification before I continue. We can proceed with anonymized examples, a narrower draft, or a manual review workflow first. Legal / compliance or account-owner confirmation may be needed depending on the material and intended use.
```

# 9. Portfolio Permission Box

Complete this before turning any job into portfolio material.

```text
Portfolio use allowed:
[ ] Public case allowed
[ ] Anonymized case allowed
[ ] Internal-only case
[ ] No portfolio use

Permission owner:

Date approved:

Allowed screenshots / assets:

Restricted details:

Can client name be shown? yes / no
Can industry be shown? yes / no / generic only
Can deliverable screenshots be shown? yes / no / blurred only
Can metrics be shown? yes / no / range only / not applicable
Can process be described? yes / no / anonymized only
```

Portfolio safety rule:

```text
Do not present fake, generic, or anonymized examples as verified real client results.
```

# 10. Minimum Viable Workflow

Use this for most small-to-medium jobs.

```text
Client material
→ safety check
→ select template
→ draft
→ human review
→ delivery
→ optional portfolio note
```

Expanded version:

```text
1. Receive material.
2. Check privacy, copyright, claims, and account-access risk.
3. Select 1 primary template and 1 to 2 supporting templates.
4. Draft using only safe or anonymized material.
5. Human reviews facts, tone, claims, privacy, and CTA.
6. Deliver in the agreed format.
7. Ask / confirm portfolio permission after delivery if appropriate.
```

One-line rule:

```text
Small job = safety check + one production template + approval + optional portfolio note.
```
