# Workflow Template Index

## Purpose

This index organizes the AI PM Radar documentation system into a practical usage map for AI PM, IP building, digital marketing, landing page / prototype planning, AI workflow audit, digital transformation consulting, and portfolio case generation.

Use this file as the entry point when deciding which template to use first.

The system follows a documentation-first workflow:

```text
Source material → triage → structured notes → repurpose → campaign / prototype / audit → portfolio case
```

The goal is not to automate everything. The goal is to turn research, client-safe notes, and project work into reusable, reviewed, and portfolio-ready artifacts.

---

# 1. 接案前 / Discovery

Use this stage before accepting a job, drafting a proposal, or using AI on any client material.

## Files to Use

### `docs/AI_RESOURCE_TRIAGE_TEMPLATE.md`

Use when evaluating a new resource, tool, article, GitHub repo, newsletter, X / Threads post, AI tool, or client reference.

Best for:

- deciding whether a resource is useful for AI PM, IP building, marketing, consulting, or portfolio work
- avoiding hype-driven tool adoption
- scoring evidence quality, cost, dependency, risk, and portfolio value
- deciding `Use / Test / Park / Reject`

Use it before adding any resource into a client workflow or AI PM Radar content pipeline.

### `docs/CONSULTING_AUDIT_TEMPLATE.md`

Use when starting a client discovery or workflow audit.

Best for:

- understanding client type, current workflow, business goal, manual tasks, and pain points
- identifying AI opportunities and safe MVP scope
- mapping 7-day and 30-day roadmaps
- producing audit findings and portfolio-safe evidence

Use it when the job is not only content writing, but also involves workflow improvement, consulting, or digital transformation.

### `docs/CLIENT_SAFE_AI_WORKFLOW.md`

Use before any client-facing AI-assisted work.

Best for:

- deciding what must not be uploaded to AI tools
- anonymizing notes
- checking approval before publishing
- controlling copyright, brand, platform account, and hallucination risk
- creating a safe fallback workflow when data sensitivity is unclear

Use it as the safety gate before processing meeting notes, interviews, screenshots, reports, transcripts, or private documents.

## Discovery Output

At the end of Discovery, produce:

```text
- client-safe source list
- workflow audit direction
- AI opportunity shortlist
- risk / sensitivity notes
- MVP recommendation candidate
- next client questions
```

# 2. 研究與資料整理 / Research & Knowledge Capture

Use this stage to turn raw input into structured notes before creating public content, campaign materials, landing pages, or portfolio cases.

## Files to Use

### `docs/IP_BUILDING_WORKFLOW.md`

Use when turning raw material into reusable IP assets.

Inputs may include:

- long-form notes
- client interviews
- course materials
- activity reports
- product research
- AI PM Radar articles
- screenshots
- PDFs
- transcripts

Outputs may include:

- Threads posts
- IG captions
- LinkedIn posts
- carousel outlines
- campaign ideas
- landing page angles
- portfolio case notes

### `docs/CONTENT_REPURPOSING_TEMPLATE.md`

Use when one source should become multiple content outputs.

Best for:

- 5 Threads post ideas
- 3 IG captions
- 1 LinkedIn post
- 1 carousel outline
- 1 landing page angle
- 1 portfolio case note

Use this after the source has been cleaned, summarized, and reviewed for privacy and claims.

### `docs/AI_ASSISTED_PM_WORKFLOW.md`

Use when the input is project management material.

Best for:

- meeting notes to action tracker
- UAT feedback to issue log
- weekly status report drafting
- requirement clarification
- risk register update
- vendor follow-up
- stakeholder-ready documentation

This file reinforces the human-in-the-loop rule: AI can draft and classify, but humans approve facts, priorities, risks, and wording.

### `docs/CONTENT_WORKFLOW.md`

Use when the source is intended for AI PM Radar daily content or source-aware research notes.

Best for:

- selecting credible sources
- writing summary, business angle, AI PM angle, and risk note
- maintaining source URL and trust scoring discipline
- avoiding unsupported claims and hype

## How Inputs Flow into Structured Notes

```text
PDF → extract relevant section → summarize → source note → risk note → content / case artifact

Screenshot → crop / blur private data → visual observation → UX / marketing insight → anonymized example

Transcript → review speaker accuracy → extract pain points / decisions → action items or content angles

Notion notes → knowledge capture → research documentation → structured brief → reusable workflow artifact

Source links → triage evidence level → summarize core idea → decide use / test / park / reject
```

## Research Output

At the end of Research & Knowledge Capture, produce:

```text
- cleaned source note
- audience pain point
- key message
- AI PM / business / marketing angle
- risk note
- possible output formats
```

Use this stage when moving from source material into deliverable content, campaign angles, CTA, landing page structure, or prototype planning.

## Files to Use

Use to decide which service package fits the job.

Best for:

- defining target clients and pain points
- choosing deliverables
- setting process boundaries

Use to convert approved source material into multi-channel output.

Best for:

- content calendar input

- landing page angle

### `docs/LANDING_PAGE_PROTOTYPE_BRIEF.md`

Use when the campaign or service needs a landing page, waitlist page, service page, or demo prototype.

Best for:

- product / service name
- target audience
- problem statement
- value proposition
- page sections
- CTA
- competitor reference notes
- content assets
- visual direction
- validation checklist
- copyright / risk note

Competitor pages should be used for structure learning only. Do not directly copy text, layout, visuals, brand assets, screenshots, code, claims, or proprietary structure.

## Execution Flow

```text
Clean source material
→ choose service package
→ define audience and pain point
→ create content angles
→ create CTA
→ plan content calendar
→ draft landing page / prototype structure if needed
→ human review
→ client approval
```

```text

- CTA
- carousel outline

- risk / claim checklist
```

# 6. Recommended Usage Sequence

## Sequence A: IP Building Client Job

Use when a client wants to turn expertise, notes, talks, screenshots, interviews, or course materials into reusable IP content.

```text
1. docs/CLIENT_SAFE_AI_WORKFLOW.md
   - Check what can and cannot be processed with AI.
   - Anonymize notes before using AI tools.

2. docs/CONSULTING_AUDIT_TEMPLATE.md
   - Understand client type, goals, current content workflow, pain points, and repeated tasks.

3. docs/IP_BUILDING_WORKFLOW.md
   - Convert raw material into reusable insight, audience pain point, and output candidates.

4. docs/CONTENT_REPURPOSING_TEMPLATE.md
   - Turn one approved source into Threads, IG, LinkedIn, carousel, landing page angle, and case note.

5. docs/DIGITAL_MARKETING_SERVICE_PACK.md
   - Package the work as an AI-assisted IP Content System.

6. docs/PORTFOLIO_CASE_TEMPLATE.md
   - Convert the approved / anonymized work into portfolio evidence.
```

Expected output:

```text
- IP theme map
- content pillars
- repurposed content drafts
- privacy / claim checklist
- portfolio-safe case note
```

## Sequence B: Digital Marketing Campaign Job

Use when a client needs campaign planning, content angle development, landing page direction, or stakeholder proposal support.

```text
1. docs/CLIENT_SAFE_AI_WORKFLOW.md
   - Confirm data safety, publishing approval, copyright, and brand risks.

2. docs/AI_RESOURCE_TRIAGE_TEMPLATE.md
   - Evaluate campaign references, competitor pages, AI tools, and source links.

3. docs/DIGITAL_MARKETING_SERVICE_PACK.md
   - Select AI-assisted Campaign Proposal or Landing Page / Demo Prototype Sprint.

4. docs/CONTENT_REPURPOSING_TEMPLATE.md
   - Convert source material into campaign messages, posts, carousel, and CTA options.

5. docs/LANDING_PAGE_PROTOTYPE_BRIEF.md
   - Define landing page structure, CTA, visual direction, competitor reference notes, validation checklist, and copyright risk.

6. docs/PORTFOLIO_CASE_TEMPLATE.md
   - Turn the campaign planning work into a case study, CV bullet, LinkedIn post, and interview pitch.
```

Expected output:

```text
- campaign brief
- target audience and pain point
- message hierarchy
- CTA
- content angle list
- landing page / prototype brief
- risk notes
- portfolio case draft
```

## Sequence C: AI PM Portfolio Case

Use when converting a self-initiated project, AI PM Radar update, prototype, or learning workflow into career evidence.

```text
1. docs/PROJECT_BRIEF.md
   - Anchor the case in AI PM Radar positioning, users, problem, MVP, and non-goals.

2. docs/CONTENT_WORKFLOW.md
   - Show source-aware research, business angle, AI PM angle, and risk note discipline.

3. docs/AI_RESOURCE_TRIAGE_TEMPLATE.md
   - Record why sources, tools, or frameworks were selected, tested, parked, or rejected.

4. docs/AI_ASSISTED_PM_WORKFLOW.md
   - Explain how AI-assisted project coordination, documentation, UAT, or stakeholder reporting was used.

5. docs/PORTFOLIO_CASE_TEMPLATE.md
   - Create the final case study, CV bullet, LinkedIn post, and 60-second interview pitch.
```

Expected output:

```text
- portfolio case note
- CV project bullet
- LinkedIn project post
- 60-second interview pitch
- GitHub / Notion portfolio write-up
```

# 7. Minimum Viable Workflow

Use this when time is limited and you need the smallest useful flow.

```text
Source material
→ docs/AI_RESOURCE_TRIAGE_TEMPLATE.md
→ docs/CONTENT_REPURPOSING_TEMPLATE.md
→ docs/DIGITAL_MARKETING_SERVICE_PACK.md or docs/LANDING_PAGE_PROTOTYPE_BRIEF.md
→ docs/PORTFOLIO_CASE_TEMPLATE.md
```

## Minimum Steps

```text
1. Triage the source.
   Decide whether to use, test, park, or reject it.

2. Clean and repurpose.
   Remove sensitive data and turn one source into content / campaign / landing page angles.

3. Create campaign or prototype direction.
   Choose a CTA, content angle, and page / campaign structure.

4. Human review.
   Check facts, privacy, claims, copyright, and client approval.

5. Portfolio case.
   Convert the work into CV, LinkedIn, interview, and GitHub / portfolio material.
```

Minimum output:

```text
- one source note
- one risk note
- one campaign / prototype angle
- one content output
- one portfolio case note
```

# 8. Risk Notes

Apply these rules across all workflows.

```text
Do not use real client data in public portfolio.
Do not upload sensitive personal data to AI tools.
Do not copy competitor content directly.
Human review is required before publishing or client delivery.
Legal / compliance confirmation may be needed depending on sector and jurisdiction.
```

## Additional Rules

- Do not paste API keys, `.env`, passwords, private keys, OAuth tokens, session cookies, or credentials into AI tools.
- Do not upload raw client documents unless the client has approved the workflow and relevant policy allows it.
- Do not use student, minor, service-user, medical, counselling, HR, legal, or financial data with AI tools unless proper approval and safeguards exist.
- Do not imply guaranteed sales, conversion, reach, productivity, or compliance outcomes.
- Do not present a prototype as a production-ready system.
- Do not use competitor pages as copy sources. Use them only for structure learning and positioning comparison.
- Keep assumptions, limitations, and human review points visible.
- For portfolio use, label fake, generic, or anonymized examples clearly.
