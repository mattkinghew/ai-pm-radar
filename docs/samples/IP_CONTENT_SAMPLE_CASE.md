# Sample Case: AI-assisted IP Content System for an Education Workshop

## Purpose

This is a fake / self-initiated sample case for demonstrating how the AI PM Radar workflow templates can support IP building, digital marketing, and portfolio case generation.

This document does not use real client data, real student data, real youth / SEN / service-user records, or private organization materials.

Use this sample to show how a small client job can move through:

```text
Client material → safety check → template selection → content repurposing → campaign direction → human review → portfolio note
```

---

# 1. Case Overview

## Scenario

A small education provider wants to promote a 2-hour introductory workshop called:

```text
AI for Everyday Learning: A Beginner Workshop for Educators
```

The provider has rough materials but no clear content system.

## Fake Client Type

- Education / training provider
- Small team
- Limited marketing budget
- Needs practical content for social media and a simple campaign page

## Main Goal

Turn scattered workshop notes into a small IP content pack and campaign direction that can support workshop promotion.

## Scope Assumption

This sample represents a small HK$1000–3000 style job.

Included:

- source material review
- safety check
- content angle planning
- short content pack
- landing page angle
- portfolio-safe case note

Not included:

- paid ad setup
- actual posting
- account operation
- full graphic design
- analytics report
- guaranteed sign-ups, reach, leads, or sales

---

# 2. Input Materials

## Fake Source Materials

```text
1. Workshop outline
2. Rough speaker notes
3. 3 screenshots of sample AI tools with no account information
4. Short description of target learners
5. Previous workshop feedback rewritten as fake examples
```

## Source Safety Check

| Material | Risk Level | Handling |
|---|---|---|
| Workshop outline | Low | Safe to summarize |
| Speaker notes | Low | Rewrite into public-friendly language |
| Tool screenshots | Medium | Remove account names, private prompts, and personal data |
| Target learner description | Low | Keep generic, no personal data |
| Feedback examples | Medium | Use fake / anonymized examples only |

## Privacy Rule

No real student names, school names, SEN status, counselling records, personal learning difficulties, or identifiable participant details are used.

If real education / youth / SEN / social-service data appears, the workflow must stop and switch to anonymized summary or fake data first.

# 5. Content Strategy

## Target Audience

Primary:

- Educators who are curious about AI but do not know where to start

Secondary:

- NGO programme staff who want to use AI for lesson preparation, activity planning, or admin support

## Audience Pain Points

```text
1. AI tools feel overwhelming.
2. Educators worry about accuracy and student privacy.
3. Many AI tutorials are too technical.
4. They want practical examples, not theory only.
5. They need safe boundaries before using AI in real education settings.
```

## Key Message

```text
AI does not replace educators. It can help educators prepare, organize, draft, and reflect faster when used with human review and privacy awareness.
```

## Content Pillars

| Pillar | Description |
|---|---|
| Practical AI literacy | Explain AI in simple education scenarios |
| Human-in-the-loop teaching | Emphasize educator judgment and review |
| Privacy-safe workflow | Avoid uploading student or sensitive data |
| Low-cost productivity | Use AI for drafts, summaries, lesson ideas, and reflection |

# 6. Repurposed Content Outputs

## 5 Threads Post Ideas

### Post 1: AI Is Not a Magic Teacher

Hook:

```text
AI 不是魔法老師，也不應該取代老師。
```

Angle:

Explain that AI is more useful as a preparation and reflection assistant than as an authority.

CTA:

```text
想開始用 AI，可以先從「整理教案」而不是「直接產生答案」開始。
```

### Post 2: 3 Safe Ways Educators Can Start Using AI

Hook:

```text
如果你是教育工作者，第一步不應該是把學生資料貼入 AI。
```

Content:

```text
1. 用假資料試教案框架
2. 用 AI 整理公開資料
3. 用 AI 產生問題清單，再由人手修正
```

CTA:

```text
安全使用，比快速使用更重要。
```

### Post 3: AI for Lesson Preparation

Hook:

```text
備課時，AI 最有用的地方不是代替你想，而是幫你開第一版。
```

Content:

```text
可以請 AI 幫你列活動流程、分級問題、學生可能誤解的地方，再由你決定是否適合課堂。
```

### Post 4: What Not to Upload

Hook:

```text
教育場景用 AI，先記住一條紅線：不要上傳可識別學生資料。
```

Content:

```text
學生姓名、相片、SEN 狀況、輔導紀錄、個案描述、家長資料，都不應直接放入 AI 工具。
```

### Post 5: Beginner Workshop Positioning

Hook:

```text
這不是一個教你追最新 AI 工具的工作坊。
```

Content:

```text
這是一個幫教育工作者建立安全、實用、可持續 AI 工作流的入門工作坊。
```

CTA:

```text
適合想開始，但不想盲目追工具的人。
```

## LinkedIn Post Draft

```text
Many education teams are interested in AI, but the real challenge is not tool selection. The challenge is building a safe and practical workflow.

For educators, AI should not start with uploading student data or asking a chatbot to produce final answers. A safer starting point is to use AI for low-risk support tasks:

- organizing lesson ideas
- drafting activity flows
- generating reflection questions
- summarizing public materials
- preparing first drafts for human review

The key principle is human-in-the-loop. AI can draft, classify, and suggest, but educators remain responsible for accuracy, context, privacy, and final decisions.

For my AI-assisted IP content workflow, I use a simple structure:

Source material → safety check → content angle → draft → human review → delivery

This approach is especially useful for education, NGO, and social-impact contexts where data sensitivity and trust matter.
```
